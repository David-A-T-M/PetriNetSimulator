## Regex Section

This project uses regex patterns to validate the structure of Petri net log entries generated by `PetriLogger`.

The patterns are centralized in `src/main/java/ar/edu/unc/david/petrinetsimulator/PetriLogPatterns.java`.

### Log format overview

The logger writes:

1. A CSV header:
   `Timestamp,Thread,Transition,MarkingBefore,MarkingAfter`
2. One CSV line per firing event (from `logFire`), for example:
   `1774364518453,Prod-0,0,"[2; 0; 0; 2; 0; 0; 3; 1; 0]","[1; 1; 0; 2; 0; 0; 2; 0; 0]"`
3. A final trailer line on close:
   `--- End of simulation: 2026-03-24T10:00:00 ---`

### Regex patterns

- **Header**
  ```regex
  ^Timestamp,Thread,Transition,MarkingBefore,MarkingAfter$
  ```
- **Firing event**
  ```
  ^\d{13},[A-Za-z0-9._-]+,\d+,"\[-?\d+(?:;\s-?\d+)*\]","\[-?\d+(?:;\s-?\d+)*\]"$
  ```
- **Trailer**
  ```
  ^--- End of simulation: .+ ---$
  ```

## Log Validation

The simulation already performs log validation from Java at the end of `main` (using the built-in validator).

As an additional verification option, you can also run the Python verifier script manually:

```bash
python3 verifier.py
```
or
```bash
python3 verifier.py --config some_config_file.json --log some_log_file.txt
```
